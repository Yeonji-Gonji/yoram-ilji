# 개발 리포트: 정보처리기사 실기 CBT 기능 (/quiz)

> 작성일: 2026-07-14 · 리포: yoram-ilji · 목적: 2026-07-19 정보처리기사 실기 대비 개인 문제풀이 (PC·모바일)
> 레퍼런스: 정처기 감자(jeongcheogi.edugamja.com) 스타일의 기출·유형별 학습 UX

## 1. 개요

시나공 기출 PDF 7종과 웹수집 기출복원 문제를 데이터화해 사이트 내에서 모의고사·유형별 연습·오답노트로 풀 수 있는 CBT 기능. 검색엔진 비노출(noindex), 개인 학습 전용.

- 웹수집 기준: **답이 2개 이상 독립 출처에서 일치하거나 코드 트레이스로 직접 검증 가능한 문제만 수록**. 출처 상충·코드 불완전·지문 미복원 문제는 제외 (제외 사유는 수집 에이전트 로그에 기록). 각 문제에 `sourceUrl` 보존
- 그림 필수 문제는 `figure: true` + 원본 PDF 페이지 링크로 폴백

## 3. 기능

- **/quiz**: D-day(7/19), 총·해결·오답 통계, 유형별 진행률(17개 카테고리), 최근 모의고사 성적
- **/quiz/exam**: 출제 범위(카테고리 다중 선택)·문항 수(10/20/30) 설정 → 타이머·문제 바로가기·이전/다음 → 일괄 채점(문제당 동일 배점, 100점 환산, 60점 합격선 표시) → 오답 자동 오답노트 적재
- **/quiz/practice**: 한 문제씩 즉시 정답·해설 확인, "맞힌 적 있는 문제 제외" 토글, 세션 정답률
- **/quiz/review**: 오답 모아보기(틀린 횟수·최신순), 인라인 재도전, 맞히면 목록에서 제거
- **채점**: short 유형 자동 채점(공백·대소문자·따옴표·문말 세미콜론 정규화, 한/영 표기 aliases 허용) + 서술형·그림 문제는 "맞았음/틀렸음" 자가 채점(점수 실시간 반영)
- **기록**: localStorage (`quiz-wrong-notes`·`quiz-exam-history`·`quiz-solved-ids`), 기기별 저장
- **반응형**: 모바일 390px 기준 검증, 코드 블록 가로 스크롤, 지문 markdown 표 렌더링

## 5. 검증

1. **추출 표본 검증**: 키워드 1~13번·SQL 1번을 원본 PDF와 수동 대조 → 일치
2. **전수 검증 워크플로우** (멀티에이전트 11파트 병렬): 340문제 전부에 대해 PDF 정답 대조 + 코드 독립 재트레이스 → **채점에 영향 주는 확정 불일치 0건**
3. **빌드**: `next build` 통과 (/quiz 4개 라우트 생성)
4. **브라우저 스모크**: 데스크톱·모바일에서 응시→제출→채점→오답노트 흐름 확인
5. **수동검증(QA)**: feature-verifier 별도 보고서 → `docs/archive/report-manual-verify-2026-07-14.md`

## 6. 유의사항 / 남은 것

- **원본 PDF 삭제 대응 (07-14)**: 시나공 PDF는 `public/`에서 삭제됨. 그림 필수 문제 5건(keyword 007 패키지 다이어그램·014 E-R·040 구조도·060 순서도, past-2026-1-03 DB설계 순서도)은 **인라인 SVG(`figureSvg`, currentColor 테마 대응)로 재현해 데이터에 내장**, "원본 PDF 보기" 링크는 제거. keyword-060은 figureDesc가 정답 경로와 불일치(③→②, ⑥→②로 오기)해 정답 정합 흐름(③→④, ⑥→①)으로 교정
- 기출복원 5문제는 원본 표·다이어그램 자체가 미복원이라 SVG 재현 불가: 텍스트 설명 박스로 표시, 답은 교차 출처 일치로만 검증(`figure: true`)
- 기록이 localStorage 기반이라 기기 간 동기화 없음 (시험까지 5일, 단일 기기 사용 전제)
- 제작 과정: PDF 추출 8에이전트 + 웹수집 회차별 7에이전트(2중 출처 교차검증) + 검증 워크플로우 22에이전트 병렬 운용
