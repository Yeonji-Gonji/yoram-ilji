## 항목별 증거

### 1. /quiz 대시보드: PASS

- "시험까지 D-5", 제목, "총 340문제 · 해결 N문제 · 오답 N개" 렌더.
- 모드 카드 3종(모의고사 → /quiz/exam, 유형별 연습 → /quiz/practice, 오답노트 → /quiz/review) 링크 정상.
- 유형별 진행률 17개 카테고리(키워드·이론 130 ~ 기출 2026년 1회 17) 렌더, 문항 수 합계 = 340 일치.
- "최근 모의고사" 기록 카드 표시. localStorage 기반 카운트가 세션 활동을 반영해 갱신됨(해결 1 → 4).
- 모바일: 카드 세로 스택, 진행률 바 정상, 페이지 가로 오버플로 없음(scrollWidth 390 = viewport). 증거: `quiz-dashboard-mobile.png`

### 2. /quiz/exam 모의고사: PASS

- 설정 화면: 출제 범위 칩 17개, 문항 수 10/20/30 버튼, "실제 실기 시험은 20문제 · 150분" 안내 문구.
- 10문제 시작 → "1 / 10" 진행 표시, 타이머 증가(00:00 → 00:24 → 01:03) 확인.
- 답 입력: 텍스트 입력 정상. 문제 이동(다음/이전) 후에도 입력값 보존("테스트 답안 1" 유지).
- 이전 버튼은 1번 문제에서 비활성화. 1~10 바로가기 버튼으로 5번 문제 점프 정상.
- 제출: confirm 다이얼로그("안 푼 문제가 7개 있습니다. 제출할까요?") 표시 → 수락 시 채점.
- 채점 결과: "10점 / 10문제 중 1개 정답 / 불합격권 (60점 미만)" + 문제별 정답/오답 뱃지 + 정답·허용 답·해설 표시. short 자동 채점 정상(문제 2에 "10" 입력 → 정답 처리).
- 자가 조정: 오답 문제의 "맞았음" 클릭 → 10점 → 20점(2개 정답)으로 즉시 변동, "틀렸음" 클릭 → 10점으로 복귀. 양방향 확인.
- 모바일: 다중 빈칸(①/②) 문제에 3, 5 입력 → 자동 채점 정답 처리(다중 빈칸 채점 정상). confirm·채점 결과 화면 모두 정상, 오버플로 없음. 증거: `quiz-exam-mobile.png`

### 3. /quiz/practice?category=sql: PASS

- SQL 카테고리 진입 시 "SQL / 1 / 17" 표시, 지문 표(<학생>, <실행결과>) 렌더 정상.
- 정답 입력(`SELECT 학과, COUNT(*) AS 학과별튜플수 FROM 학생 GROUP BY 학과;`) → "정답 확인" → 정답 처리 + 해설 노출 + 세션 카운터 "1문제 중 1개 정답" 갱신 + "맞았음/틀렸음" 자가 조정 버튼 표시.
- "다음 문제" → 2/17 진행 정상.
- "맞힌 적 있는 문제 제외" 토글 ON → 풀이 17 → 16으로 감소, 라벨 "(현재 1문제 해결)" 갱신. 방금 맞힌 문제가 실제로 제외됨.

### 4. /quiz/review 오답노트: PASS

## 발견 이슈 (경미 3건, 블로킹 없음)

1. **채점 허용 답 데이터 보강 여지 (데이터 레벨)**
   - 재현: /quiz/exam에서 디자인 패턴 문제(기출 2025년 1회, 래퍼 패턴)에 "어댑터" 입력 → 제출.
   - 결과: 허용 답이 `adapter / 어댑터 패턴 / Adapter`뿐이라 "어댑터" 단독 입력은 오답 처리.
   - 영향: "맞았음" 자가 조정 버튼으로 사용자 복구 가능. 허용 답 목록에 한글 단독 표기 추가를 권장.

2. **연습 화면 "제외" 토글 라벨의 해결 수 갱신 지연 (UI)**
   - 재현: /quiz/practice에서 문제를 맞힌 직후 토글 라벨 확인.
   - 결과: 방금 맞힌 문제가 있는데 라벨은 "(현재 0문제 해결)"로 표시되다가 토글 조작(재렌더) 시점에 "(현재 1문제 해결)"로 갱신됨. 제외 동작 자체는 정확함.

3. **모바일 390px에서 "전체 비우기" 버튼 텍스트 2줄 줄바꿈 (오답노트 헤더, 시각적)**
   - 재현: 390px에서 /quiz/review 진입. `quiz-review-mobile.png` 우상단 참조.
   - 영향: 기능 문제 없음. `whitespace-nowrap` 등으로 개선 가능.

> 참고(검증 환경 노이즈, 퀴즈 기능 아님): 검증 중 병렬 세션의 Header.tsx 편집으로 일시적 컴파일 에러·500이 콘솔에 남았음. 최종 파일은 정상이며 재현되지 않음.

## 사용자 직접 검증 안내

dev 서버(http://localhost:3000)는 그대로 살아 있습니다. 브라우저에서 이어서 확인하려면:

1. `http://localhost:3000/quiz` 접속 → D-day·총 문제 수·모드 카드 3종·진행률 17개 확인.
2. 모의고사: 범위 칩 몇 개만 선택해 시작 → 선택 범위의 문제만 나오는지 확인(이번 검증은 전체 범위만 수행). 20문제·30문제 모드, 타이머 장시간 동작, "새 모의고사" 버튼도 미검증 항목.
3. 연습: `?category=keyword` 등 다른 카테고리, essay(자가 채점 전용) 유형 문제의 "맞았음/틀렸음" 흐름 확인(이번 검증은 short 자동 채점 중심).
4. 오답노트: "전체 비우기" 버튼(이번 검증에서는 기록 보존을 위해 미클릭), 재도전 오답 시 목록 유지되는지 확인.
5. localStorage 초기화 테스트: 개발자도구 → Application → Local Storage에서 quiz 관련 키 삭제 후 빈 상태(오답 0, 기록 없음) 렌더 확인.
6. 주의: 이 검증 세션에서 실제 응시 기록 2건·해결 문제·오답이 localStorage에 남아 있습니다. 실사용 전 초기화를 원하면 위 5번 절차 수행.
